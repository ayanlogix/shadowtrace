System Architecture

ShadowTrace leverages the Midnight Network's Dual-State model to enforce zero-trust identity verification.

The Private State

All sensitive user data (DOB, Financials) is processed entirely client-side inside the browser memory. The data is parsed through our Compact ZK-circuits to generate a proof and is immediately purged from RAM. It never touches a centralized server.

The Public State

Only the resulting 128-byte zk-SNARK cryptographic hash is broadcasted to the Midnight Ledger. The public network validators verify the mathematical proof, not the raw data.

SDK Reference

Integrate ShadowTrace ZK-Identity into your own Web3 DApp in just 3 lines of code.

TerminalTypeScript
import { ShadowTraceProver } from '@shadowtrace/sdk';

// 1. Initialize Prover with User Wallet
const prover = new ShadowTraceProver(walletConnector);

// 2. Request ZK Proof for Age >= 18
const proof = await prover.verifyConstraint('AGE_OVER_18');

// 3. Submit proof to on-chain smart contract
console.log("Proof Hash:", proof.hash);
